SnapshotWorkflow

Operations for virtual copy snapshots.

API Reference Home

Class Summary

The SnapshotWorkflow class manages snapshot lifecycle operations including creation, promotion (restore), retrieval, listing and deletion. REST endpoints are under /volumes, treating snapshots as thin-provisioned virtual volumes.

Methods Overview

MethodDescription

create_snapshot POST

create_snapshot(volume_name, snapshot_name, optional=None)

Create a snapshot of a base volume.

ParameterTypeRequiredDescription
volume_namestrYesName of the source volume.
snapshot_namestrYesName for the new snapshot.
optionaldictNoAdditional snapshot creation parameters.

Returns: dict - Snapshot creation response.

Endpoint: POST /volumes/{volume_name} (action=createSnapshot)

promoteVirtualCopy PUT

promoteVirtualCopy(name, params)

Promote (restore) a snapshot to replace its parent volume.

ParameterTypeRequiredDescription
namestrYesName of the snapshot to promote.
paramsdictNoOptional promotion parameters (e.g., retry flags).

Returns: dict - Promotion operation result.

Endpoint: PUT /volumes/{name} (action=promoteVirtualCopy)
