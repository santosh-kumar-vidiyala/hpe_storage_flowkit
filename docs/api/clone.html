CloneWorkflow

Physical copy (clone) operations for volumes.

API Reference Home

Class Summary

The CloneWorkflow class provides operations for creating a physical copy (clone) of a volume and controlling its lifecycle including stopping or resynchronizing the copy.

Methods Overview

MethodDescription
copyVolumeInitiate a physical copy from a source volume.
stopOfflinePhysicalCopyStop an offline physical copy operation.
resyncPhysicalCopyResynchronize a physical copy with its source.

copyVolume POST

copyVolume(src_name, info)

Initiate a physical copy from a source volume to a destination volume.

ParameterTypeRequiredDescription
src_namestrYesName of source volume.
infodictYesPayload including destination name and copy options.

Returns: dict - Copy initiation response.

Endpoint: POST /volumes/{src_name}

stopOfflinePhysicalCopy PUT

stopOfflinePhysicalCopy(name, info)

Stop an ongoing offline physical copy operation.

ParameterTypeRequiredDescription
namestrYesName of the clone volume (or source context).
infodictYesPayload containing action stop code.

Returns: dict - Stop operation result.

Endpoint: PUT /volumes{name}

resyncPhysicalCopy PUT

resyncPhysicalCopy(volume_name, info)

Resynchronize a previously created physical copy.

ParameterTypeRequiredDescription
volume_namestrYesName of the clone volume.
infodictYesPayload with resync action code.

Returns: dict - Resync operation response.

Endpoint: PUT /volumes/{volume_name}
