VolumeWorkflow

Operations for Virtual Volumes.

API Reference Home

Class Summary

The VolumeWorkflow class provides CRUD and lifecycle operations for volumes, including growth and tuning. Methods map directly to REST endpoints under /volumes.

Methods Overview

MethodDescription
create_volumeCreate a new virtual volume.
modify_volumeModify existing volume attributes.
grow_volumeIncrease capacity of a volume.
tune_volumeTune provisioning/performance settings.
get_volumeRetrieve a volume's details.
list_volumesList all volumes.
delete_volumeDelete a volume.

create_volume POST

create_volume(name, cpg, sizeMiB, params=None)

Create a new virtual volume.

ParameterTypeRequiredDescription
namestrYesVolume name.
cpgstrYesBacking CPG name.
sizeMiBintYesSize of volume in MiB.
paramsdictNoAdditional attributes (compression, provisioning, etc.).

Returns: dict - Created volume resource.

Endpoint: POST /volumes

modify_volume PUT

modify_volume(name, volume_mods, app_type=None)

Modify volume attributes.

ParameterTypeRequiredDescription
namestrYesVolume name.
volume_modsdictYesFields to update.
app_typestrNoOptional application classification (metadata).

Returns: dict - Updated volume resource.

Endpoint: PUT /volumes/{name}

grow_volume PUT

grow_volume(name, new_size)

Increase the size of an existing volume.

ParameterTypeRequiredDescription
namestrYesVolume name.
new_sizeintYesNew total size in MiB after growth.

Returns: dict - Growth operation result.

Endpoint: PUT /volumes/{name} (action=3)

tune_volume PUT

tune_volume(name, info)

Tune provisioning or performance attributes of a volume.

ParameterTypeRequiredDescription
namestrYesVolume name.
infodictYesPayload with tuning parameters (e.g., action codes, policies).

Returns: dict - Tuning operation result.

Endpoint: PUT /volumes/{name}
