CPGWorkflow

Operations for Common Provisioning Groups (CPGs).

API Reference Home

Class Summary

The CPGWorkflow class provides CRUD operations for Common Provisioning Groups (CPGs). Each method maps directly to a REST endpoint on the storage array.

Methods Overview

MethodDescription
create_cpgCreate a new CPG with optional attributes.
delete_cpgDelete an existing CPG by name.
get_cpgRetrieve details of a specific CPG.
list_cpgsList all CPGs on the array.

create_cpg POST

create_cpg(name, params=None)

Create a new CPG.

ParameterTypeRequiredDescription
namestrYesName of the CPG.
paramsdictNoAdditional CPG attributes (e.g. RAID type, growth settings).

Returns: dict - REST API response payload.

Endpoint: POST /cpgs

delete_cpg DELETE

delete_cpg(name)

Delete an existing CPG.

ParameterTypeRequiredDescription
namestrYesName of the CPG to delete.

Returns: dict | None - REST API response or None if no body.

Endpoint: DELETE /cpgs/{name}

get_cpg GET

get_cpg(name)

Retrieve details of a CPG.

ParameterTypeRequiredDescription
namestrYesName of the CPG.

Returns: dict - CPG details.

Endpoint: GET /cpgs/{name}

list_cpgs GET

list_cpgs()

List all CPGs.

ParameterTypeRequiredDescription
---No parameters.

Returns: list[dict] - Array of CPG summaries.

Endpoint: GET /cpgs
